# storytelling — 事実→ナラティブ変換スキル

事実を正確に保ったまま、引き込まれる物語に変換する。**変えるのは伝え方であって、内容ではない。**

## いつ使うか
- 事実・データ・専門知識を「ストーリーにして」「面白く伝えて」の依頼
- 解説をナラティブ仕立てにしたいとき

## 入力（不足していれば必ず確認する）
- **事実**（事実・データ・専門的な内容）。空なら変換できないので必ず確認する。

## ルール（厳守）
- 定義からではなく、具体的なシーン・人物・瞬間から始める
- 事実をストーリーアークに織り込む: **設定 → 緊張 → 解決**
- 複雑な概念を直感的に理解させるアナロジーを使う
- 読者の理解が変わる「なるほど」の瞬間を1つ含める
- 読者の人生に結びつくテイクアウェイで締める

## 絶対条件
事実は正確なまま。脚色で内容（数字・因果・主張）を歪めない。
不確かな点は創作で埋めず、元の事実の範囲内で描く。

詳しい技法は `references/narrative-craft.md`、構成の型は `assets/story-arc-template.md` を参照。
