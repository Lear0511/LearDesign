# {スキル名}

> このファイルは常に読み込まれる。役割と基本ルールだけを書き、詳細は `references/` に逃がす。500行以下を推奨。

## 役割
このスキルが担う **1つの責務** を1〜2文で書く。複数の責務を1スキルに詰めない。

## いつ使うか
- 条件1（例: ユーザーが◯◯を依頼したとき）
- 条件2
- 使わない場面も明記すると誤発火を防げる

## 基本ルール
1. 最初に守るべき制約
2. 出力フォーマットの原則
3. やってはいけないこと

## ワークフロー
1. 必要に応じて `references/` から知識・制約・品質基準を読む
2. 繰り返し処理やデータ変換は `scripts/` を実行する
3. 出力は `assets/` のテンプレートに沿って生成する

## ディレクトリの役割
| ディレクトリ | 読み込み | 用途 |
|---|---|---|
| `SKILL.md` | 常に | 役割と基本ルールだけ（500行以下推奨） |
| `references/` | タスクに応じて | 知識・制約・品質基準 |
| `scripts/` | 実行 | 実行可能コード。繰り返し処理やデータ変換 |
| `assets/` | 出力時 | 出力用テンプレートやアイコン |

詳しい書き方は `references/AUTHORING_GUIDE.md` を参照。
