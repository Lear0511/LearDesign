# スキル作成ガイド

このファイルは `references/` の例。タスクに応じて読み込む詳細知識をここに置く。

## ディレクトリ構成

```
skill-name/
  SKILL.md         → 常に読み込む。役割と基本ルールだけ (500行以下推奨)
  references/      → タスクに応じて読み込む。知識・制約・品質基準
  scripts/         → 実行可能コード。繰り返し処理やデータ変換
  assets/          → 出力用テンプレートやアイコン
```

## それぞれの役割と書き方

### SKILL.md（常に読み込む）
- frontmatter の `name` と `description` が最重要。`description` はトリガー条件そのもの。
  「いつ・何のために使うか」を具体的に書く。曖昧だと適切な場面で発火しない。
- 本文は役割・いつ使うか・基本ルール・ワークフローだけ。
- 詳細手順や大きな知識ベースは書かず、`references/` に逃がす。
- 目安は500行以下。長くなったら分割のサイン。

### references/（タスクに応じて読み込む）
- 知識・制約・品質基準・チェックリスト・用語集など。
- 1ファイル1テーマに分ける。SKILL.md から「必要なときに参照」と指示する。
- 全部を常時読ませない。必要なものだけを名指しで読ませることでコンテキストを節約する。

### scripts/（実行可能コード）
- 繰り返し処理・データ変換・検証など、コードで確実にやるべき処理。
- LLM に毎回手作業させるより、スクリプト1本で決定的に処理する。
- 入出力と実行方法を冒頭コメントに書く。

### assets/（出力用テンプレート）
- 出力の雛形、アイコン、定型フォーマットなど。
- 生成物の体裁を揃えたいときに参照させる。

## 品質チェック
`scripts/validate_skill.py` でSKILL.mdの存在・frontmatter・行数を検証できる。

```bash
python3 .claude/skills/skill-template/scripts/validate_skill.py .claude/skills/<skill-name>
```
