# ケーススタディの書き方

## 全体トーン
- 現在形で書き、読者をその場に置く（「〜した」より「〜している」）。
- 主役はクライアント。製品/サービスは課題を解く道具として描く。

## 1. 課題（2段落）
- 1段落目: 状況と痛みを具体的に。誰が何に困り、何を失っていたか。
- 2段落目: その痛みが放置されるとどうなるかの緊張感。
- 読者が「うちと同じだ」と感じる具体描写を入れる。

## 2. アプローチ（3〜4段落）
- 何を・どの順で・なぜそうしたかを手順で書く。
- 一般論（「最適化した」）を避け、実際の動き（「◯◯を週次で計測し直した」）で書く。
- 失敗や調整も入れるとリアリティが出る。

## 3. 結果（1〜2段落）
- Before/After を数字で対比する（例: 離脱率 38% → 12%）。
- 数字は最低3つ。期間・割合・絶対値を混ぜると説得力が増す。
- 数字は提供された事実のみ。推測値は使わない。

## 4. 教訓（1段落）
- この事例から読者が自分に当てはめられる学びを1つ。
- 「だから読者は何をすべきか」に接続する。

## 5. プルクォート
- 結果に裏打ちされた、クライアントが実際に言いそうな一言。
- 宣伝臭い決め台詞ではなく、現場の本音のトーンで。
- NG:「導入して本当に良かったです！」/ OK:「毎朝ダッシュボードを開く時間が、不安じゃなくて楽しみになった。」
